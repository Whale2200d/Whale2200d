### 7. 종합 평가 및 후속 액션 제안

#### 7.1 결론

- 기존 repo에서는 **쿼리/스크롤/로직 수준**의 성능 최적화(TanStack Query 적용, 무한 스크롤, scroll fix 등)가 이루어졌고, 이 로직이 신규 repo의 `web/src/wallet/...` 구조로 **이관·재구성**되며 QueryClient 옵션·queryKey·enabled 조건 등으로 한 단계 더 최적화되었다.
- 신규 repo에서는 **이미지(WebP)와 Web Vitals 지향 UX 최적화**가 더해져, LCP·INP·CLS가 **산업 표준(LCP ≤ 2.5s, INP ≤ 200ms, CLS ≤ 0.1)**을 여유 있게 충족하는 상태(T3)에 도달한 것으로 추정된다.
- 이관 직후(T0) 대비 최종(T3)까지 **LCP 약 37%, INP·CLS 약 33% 수준의 성능 향상**을 기대할 수 있는 히스토리로 해석할 수 있다.

#### 7.2 후속 액션(권장)

- **실제 Web Vitals 측정 파이프라인 구축**  
  `web-vitals` 패키지 또는 자체 RUM 스크립트로 Git SHA 단위 LCP/INP/CLS 수집·대시보드화. 본 보고서 추정치와 실측치 비교·보정으로 성능 최적화 ROI 분석에 활용.
