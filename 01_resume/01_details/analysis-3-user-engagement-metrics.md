# 3. 사용자 참여 메트릭 연계 분석

## 사용자 참여 산업 표준 (Industry Standards)

사용자 참여 메트릭은 웹 앱에서 세션 시간, 클릭률, 유지율 등을 통해 사용자 상호작용을 측정합니다. 산업 표준은 **Google Analytics 4(GA4)**나 **Amplitude** 같은 도구로 추적되며, SaaS/웹 앱에서 **55–65%의 참여율**이 건강한 수준으로 간주됩니다. 주요 벤치마크는 다음과 같습니다:

| 메트릭 | 설명 | 산업 표준 (좋은 성능 기준) | 출처 |
|--------|------|---------------------------|------|
| Engagement Rate | 참여 세션 비율 (GA4 기준) | ≥ 55–65% | GA4 benchmarks |
| Average Session Duration | 평균 세션 시간 | ≥ 2–3분 (웹 앱 기준) | GA4 & industry data |
| Bounce Rate | 이탈률 | ≤ 40–50% | HubSpot & GA4 |
| Pages per Session | 세션당 페이지 수 | ≥ 2–3페이지 | Zendesk & industry |
| Net Promoter Score (NPS) | 추천 의향 점수 | ≥ 50 (SaaS 기준) | Zendesk & Gainsight |
| Daily/Weekly/Monthly Active Users (DAU/WAU/MAU) | 활성 사용자 비율 | DAU/MAU ≥ 20–30% | Gainsight & UXCam |

이 메트릭은 제품 성장과 직접 연계되며, **53%의 사용자**가 3초 이상 로딩 시 이탈하므로 성능과 결합해 관리하는 것이 좋습니다.

---

## 실제 프로젝트 결과

Engagement Rate·Bounce Rate·NPS는 Git만으로 측정 불가. 참여·UI·모듈화 관련 커밋을 **대리 지표**로 산업 표준 대비 방향만 추정함.

| 메트릭 | 산업 표준 | 프로젝트(대리 지표) | 산업 대비 |
|--------|-----------|----------------------|-----------|
| Engagement Rate | ≥ 55–65% | 분리 전 월 7~247건 UI·모듈화 커밋, 분리 후 wallet 315·ui 78·util 50건 | 소폭 상승 추정 (실측 없음) |
| Bounce Rate | ≤ 40–50% | 레이아웃·스크롤·PWA·접근성 개선 커밋 | 소폭 하락 추정 (실측 없음) |
| NPS | ≥ 50 | UI 일관성·한글 폰트·버튼/레이아웃 개선 | 개선 여지 (실측 필요) |

- **요약**: 분리 전 core·route·컴포넌트 분리 등 구조 개선 지속. 분리 후 PWA·접근성·일관성 강화. **정량 목표 달성 여부는 GA/설문 실측 필요.**

## 근거 1: 분석 대상 및 제약

- **대상**: `frontend-mono`(2024-03 ~ 2025-08), 분리 후 `project__front__wallet`, `module__front__ui`, `module__front__util`.
- **참여 관련 키워드**: UI, refactor, feat, 모듈, module, onboarding, layout, 개선, 경험(experience), 세션 등 커밋 메시지 기반.
- **제약**: GA·Mixpanel·NPS 설문 등 **실제 참여 메트릭 없음** → Engagement Rate·Bounce Rate·NPS 수치 불가. **기능/코드 변경 건수**와 **변경 성격(UI·모듈화·일관성)**으로 “예상 참여 변화”만 서술.

## 근거 2: 분리 전(frontend-mono) 참여 관련 변경 추이

- **참여·UI·모듈화 관련 커밋** (메시지 grep): 월별 건수 아래 표 참조.
- **모듈화·구조 개선 사례**:  
  - core 패키지 독립 라이브러리화, 브릿지 분리 준비  
  - 로딩/스플래시 프로젝트별 분리  
  - route·컴포넌트·api·쿼리 분리 (레퍼럴, 주문, 시장, 차트 등)  
  - 레이아웃·스크롤·상태 처리 개선 (onActivated, viewport 기준 등)
- **해석**: “멀티 서비스 모듈화 + UI/경험 개선”이 지속되어, **안정성·일관성 향상 → 세션 시간 증가·이탈 감소**에 유리한 방향으로만 추정 가능.

## 근거 4: 산업 표준과의 연계 및 예상 참여 변화

| 지표 | 산업 권장 | 실측 가능 여부 | Git 기반 예상 |
|------|-----------|----------------|----------------|
| Engagement Rate | ≥55–65% | 불가 | 모듈화·UI 일관성·로딩/상태 개선 → **세션·재방문에 유리** → 소폭 상승 추정 |
| Bounce Rate | ≤40–50% | 불가 | 레이아웃·스크롤·온보딩·PWA 개선 → **이탈 감소에 유리** → 소폭 하락 추정 |
| NPS | ≥50 | 불가 | 일관된 UI·접근성(한글 폰트 등)·안정성 개선 → **만족도 개선 여지** (정량화는 실측 필요) |

- **예상 참여 변화 요약**:  
  - **모듈화·공통 UI** → 서비스 간 일관성·품질 안정화 → **세션 시간·재방문 증가 가능성**.  
  - **PWA·레이아웃·접근성** → 이탈 요인 완화 → **Bounce Rate 감소 가능성**.  
  - **기능·UI 개선 지속** → **NPS 상승 가능성** (실측으로 검증 필요).

## 요약

- **제약**: 실제 Engagement Rate·Bounce Rate·NPS 없음 → Git만으로는 정량 목표 달성 여부 불가.
- **가능한 분석**: (1) 참여·UI·모듈화 관련 **커밋 건수**의 시계열, (2) **분리 전/후** 활동이 새 repo로 이전·유지되는지, (3) 변경 성격(모듈화·UI 일관성·PWA·접근성)에 따른 **예상 참여·이탈·만족도** 방향 서술.
- **산업 표준**: “해당 패턴이 Engagement Rate ↑·Bounce Rate ↓·NPS ↑에 유리하다”는 해석만 가능하며, 수치 비교는 **실측 데이터 확보 후** 진행하는 것이 타당함.
