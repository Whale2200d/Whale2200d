## 1. 요약

<!-- 한 페이지 이내로, 6개 영역의 핵심 결과와 종합 메시지를 요약 -->

### 1.1 분석 개요

- **데이터 소스**: Git 히스토리 기반 (실측 메트릭 없음 → 대리 지표로 추정)
- **주요 변화**: 모노레포 → 앱별 레포 분리(wallet·ui·util), FSD·Turbo·CI 도입, 리팩터/모듈화 지속

### 1.2 영역별 요약 (산업 표준 대비)

| 영역 | 산업 표준 요약 | 프로젝트 결과 한 줄 | 링크 |
|------|----------------|---------------------|------|
| 웹 성능 KPI | LCP≤2.5s, INP≤200ms, CLS≤0.1 | 예상 개선 LCP 5~12%, INP 10~20%, CLS 5~10% (실측 없음) | [링크](analysis-2-web-performance-kpi.md) |
| 사용자 참여 | Engagement≥55–65%, Bounce≤40–50%, NPS≥50 | 방향만 추정, GA/설문 실측 필요 | [링크](analysis-3-user-engagement-metrics.md) |
| 코드 효율성 | CC≤10, MI≥65, Coverage≥80% | 추정 58/100, MI 62 근접, Coverage 미측정 | [링크](analysis-4-code-efficiency.md) |
| 배포·스케일링 | DORA 일일 배포, Lead Time<1일, MTTR<1h | 일일 배포 가능 수준, MTTR·CFR 측정 불가 | [링크](analysis-5-deployment-scaling.md) |
| 팀/프로세스 | Velocity 20–50, Cycle Time<1주, WIP≤5–10 | Velocity 목표 충족, 분리 후 WIP 9개로 목표 근접 | [링크](analysis-6-team-process.md) |
| 비즈니스 영향 | ROI≥2x, 성장≥20%/분기, NPS≥50 | 65/100, 비용·효율 개선 유추, 실측 도입 시 조정 | [링크](analysis-7-business-impact.md) |

### 1.3 종합 메시지

- Git 히스토리 기반으로 6개 영역을 분석한 결과, 실측 메트릭 없이 **대리 지표**로만 추정 가능하여 산업 표준 대비 **정량 달성 여부는 판단 불가**하다. 다만 성능·참여·코드·배포·프로세스·비즈니스 전반에서 **목표에 유리한 변경**(캐싱·모듈화·Turbo·CI·리팩터 등)이 이어졌고, **실측(RUM·GA·배포 로그·NPS·Coverage)** 도입 시 목표 관리와 점수 정교화가 가능하다.

## 2. 웹 성능 KPI

**산업 표준**: Core Web Vitals (LCP ≤2.5s, INP ≤200ms, CLS ≤0.1), FCP ≤1.8s, TTI ≤3.8s · [링크](analysis-2-web-performance-kpi.md)

### 2.1 산업 표준 대비 프로젝트 결과

| 메트릭 | 산업 표준 | 프로젝트(대리 지표) | 산업 대비 |
|--------|-----------|----------------------|-----------|
| LCP | ≤ 2.5초 | 성능·쿼리 관련 커밋 214건, 캐싱·refetch 제거·로딩 개선 | 예상 개선 **5~12%** (실측 없음) |
| INP | ≤ 200ms | 쿼리/로딩 정리·웹소켓 전환 패턴 | 예상 개선 **10~20%** (실측 없음) |
| CLS | ≤ 0.1 | 스켈레톤·로딩 UI 추가 패턴 | 예상 개선 **5~10%** (실측 없음) |

**요약**: frontend-mono 성능·쿼리 관련 214건, 쿼리 파일 변경 371건. 실제 달성 여부는 RUM/Lighthouse 실측 필요.

| 메트릭 | 산업 표준 (DORA) | 프로젝트(대리 지표) | 산업 대비 |

## 7. 비즈니스 영향

**산업 표준**: ROI ≥2x, Business Value ≥20% 성장/분기, CSAT/NPS ≥50 · [링크](analysis-7-business-impact.md)

### 7.1 산업 표준 대비 프로젝트 결과

| 메트릭 | 산업 표준 | 프로젝트(대리 지표) | 산업 대비 |

| ROI | ≥ 2x | 수익·투자 미측정. 모듈화·중복 제거·배포 단축 | 비용 **25~35%** 절감 유추 → ROI 상승 유리 |
| Business Value | ≥ 20% 성장/분기 | 파일 1,475→2,654(약 80%↑), 앱별 독립 배포 | 확장·성장 가능성 유리 (실측 없음) |

**요약**: 6,425커밋·1,132 머지·분리 후 wallet 531커밋·1,930 파일. 중복 제거·모듈화로 25~30% 비용 절감 유추, 프로세스 15~25% 효율 추정. 개선 제안은 실측·품질·WIP·문서 강화.
